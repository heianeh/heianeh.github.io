INSERTION SORT

Posted on 2018-11-08 | In sort algorithm

Basic Idea

Insertion sort is a simple sorting algorithm. Assuming that we get one digit at a time instead of getting all numbers at first. For me, I would insert the new one to the existing sorted list once I receive it. So the basic idea of insertion sort is that insert the next elements into the sorted list one by one.

Analysis

As discussed in the Basic Idea, we receive one element once, so that we need to receive n times (Actually n - 1 times, the first elements can be ignored.) (line 5).

For each element, we should find the proper index in the numbers list (line 6-10). Because we have to insert the new element into the front list, so all the elements which are greater than the new elements have to move back one step, line 10 works for these.

Line 11: insert the new elements into the right position.

Example

Example list of numbers:

[5, 9, 2, 4, 7, 1]

For the first receive:

  1. i = 1
    newNum = 9
    j = 1
    numbers[j - 1] = 5
    numbers [5, 9, 2, 4, 7, 1]

But numbers[0] < newNum, so the numbers list:

[5, 9, 2, 4, 7, 1]

  1. i = 2
    newNum = 2
    j = 2
    number[j - 1] = 9

newNum < number[j - 1] so that j = j - 1 = 1, numbers[j] = numbers[j - 1]. Then

  1. i = 2
    newNum = 2
    j = 1
    number[j - 1] = 5
    numbers [5, 9, 9, 4, 7, 1]

newNum < number[j - 1] so that j = j - 1 = 0, numbers[j] = numbers[j - 1]. Then

  1. i = 2
    newNum = 2
    j = 0
    number[j - 1] = out of bounds
    numbers [5, 5, 9, 4, 7, 1]

now j is smaller than 1 so the numbers list:

[2, 5, 9, 4, 7, 1]

Continue to do the same process and the numbers will change as following:

[2, 4, 5, 9, 7, 1]
[2, 4, 5, 7, 9, 1]
[1, 2, 4, 5, 7, 9]
Finished

Complexity

We do n receive, for each receive, we do 1 to (n - 1) comparisons to find the proper position.
So for the best situation (the list is already sorted as we want), the time complexity is O(n)
For the worest case(the list is reversed as the final result), the time complexity is O(n^2)
For average case, do n / 2 comparisons for each receive, so the time complexity is O(n^2)
Obviously, the space complexity is constant. T(1)

SELECTION SORT

Posted on 2018-11-08 | In sort algorithm

Basic Idea

Selection sort is the most intuitive method and just like human do sorting. For me, if someboday ask me to sort a list of data, I would choose the smallest from the remaining data one by one. This is the core idea of selection sort.

Pseudo Code

Code

Analysis

As discussed in the Basic Idea, we select one smallest element once, so that we need to do selection n times (line 5).

For each selection, we have to find the smallest one. Line 6 assumes that the smallest element locates at the first index of the remainning list.

Line 7 go through the following numbers

and line 8 check if the number is smaller than the assumed smallest elements.

If so, line 9 set the smallest index to new index.

Line 10-12 swap the smallest element with the first element in the remainning list.

Example

Example list of numbers:

[5, 9, 2, 4, 7, 1]

For the first selection:

  1. i = 0
    minIndex = 0
    j = 1

Line 8 would compare numbers[1] and numbers[0] which are 9 and 5 respectively. but 9 > 5 so the condition isn’t satisfied.

  1. i = 0
    minIndex = 0
    j = 2
    number[0] = 5
    number[2] = 2

number[2] < number[0] so that minIndex = j = 2

  1. i = 0
    minIndex = 2
    j = 3
    number[2] = 2
    number[3] = 4

number[3] > number[2]

  1. i = 0
    minIndex = 2
    j = 4
    number[2] = 2
    number[4] = 7

number[4] > number[2]

  1. i = 0
    minIndex = 2
    j = 5
    number[2] = 2
    number[5] = 1

number[5] < number[2] so that minIndex = j = 5
Now we find the index of the smallest element, then swap the smallest element with the first element.
The numbers turn to

[1, 9, 2, 4, 7, 5]

Continue to do the same process and the numbers will change as following:

[1, 9, 2, 4, 7, 5]
[1, 2, 9, 4, 7, 5]
[1, 2, 4, 9, 7, 5]
[1, 2, 4, 5, 7, 9]
Finished

Complexity

We do n selection, in the first selection, we do n comparisons, and n - 1 for the second selection and so on.
So the total comparisons is n + (n - 1) + (n - 2) + … + 1 = n * (n + 1) / 2
So the time complexity is O(n^2)
Obviously, the space complexity is T(1)
