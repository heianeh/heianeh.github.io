BUBBLE SORT

Posted on 2018-11-10 | In sort algorithm

Basic Idea

Bubble sort (also known as Sinking sort) is a simple sorting algorithm which repeatly iterate the list and compare the adjcent two elements, and reverse the order if the relative order is incorrect. For each iteration (walk through the list), the smallest elements reach to head of list just like “floating” to water surface (or the largest element “sinks” to the bottom), that’s why called bubble sort.

Analysis

One largest element “sinks” down to bottom in one interation, so that for n elements, we need n interations (line 6).

In each iteration, we walk through the remaining list. Because the largest elements “sink” down, so we only need walk through the first (n - 1 - i) in ith iteration (line 7).

We compare two adjcent elements and if the relative order is incorrect, swap them (line 8-11) .

For the second pseudo code:
If there is no swap during one iteration which means the list is already sorted, the flag would still be 0. Then finish the program.

Example

Example list of numbers:

[5, 9, 2, 4, 7, 1]

In the first iteration:
First compare 5, 9, the relative order is correct.
Then compare 9, 2, swap them. Now numbers:

[5, 2, 9, 4, 7, 1]

Then compare 9, 4, swap them.

[5, 2, 4, 9, 7, 1]

Then compare 9, 7, swap them.

[5, 2, 4, 7, 9, 1]

Compare 9, 1, swap them.

[5, 2, 4, 7, 1, 9]

Continue to do the same process and the numbers will change as following:

[2, 4, 5, 1, 7, 9]
[2, 4, 1, 5, 7, 9]
[2, 1, 4, 5, 7, 9]
[1, 2, 4, 5, 7, 9]

Finished!

Complexity

If there’s no flag, bubble sort will do useless iteration even the list is already sorted. In this case, the best, worst and average time complexity is O(n2). Add a flag can reduce the time complexity of best cast to O(n). Bubble sort is not an efficient algorithm, the largest elements “sink” to the bottom while the smaller element would only move forward one position per iteration. This has led to these types of elements being named rabbits and turtles, respectively, after the characters in Aesop’s fable of The Tortoise and the Hare.
Obviously, the space complexity is constant. T(1)
