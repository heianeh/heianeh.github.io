MERGE SORT

Posted on 2018-11-14 | In algorithm

1. Basic Idea

Merge sort is a classical implementation using DIVIDE AND CONQUER. In two branch (most used) merge sort, the program recursively divide the list into two smaller size lists. After sorting two sub-list, conbine two sorted list into one large sorted list. The whole procedure has two important steps. One is divide which is pretty easy, just divide the list until the sub-list has size 1 (list with only one element is a sorted list). The other is to combine two sorted list and form a longer sorted one.

3.3 Complexity

The benefits of Divide and Conquer show in merge sort. In the divide part, we use log(n) (height of the divide tree) time to divide into single element lists. In the conquer part, for each level, we go through all elements once, so the time complexity is O(n) for each level. Then we have log(n) levels so that the total time complexity is O(n*lgn + lgn) = O(nlgn)
The space complexity S(n) in this implementation is O(n) but in fact we can reduce it to constant space O(1) (Try it yourself).
