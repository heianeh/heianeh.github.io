SHELL SORT

Posted on 2018-11-09 | In algorithm

Basic Idea

Shell sort is a variation of INSERTION SORT. In INSERTION SORT, for the best situation (already sorted as we need), it costs linear time complexity. So the purpose of Shell sort is to try to sort the list which is mostly sorted.

Detailed Idea

For insertion sort, the most important factor is the number of shifts for each iteration while shell sort tries to avoid large number of shifts. Shell sort would divide the original list into several sublist.
Original List
[a1, a2, â€¦, an]
Assuming we divide into h groups, then the elements for each group show in the following (start with index 0):

group 1: [a0, a0 + h, a0 + 2h, â€¦]
group 2: [a1, a1 + h, a1 + 2h, â€¦]
group 3: [a2, a2 + h, a2 + 2h, â€¦]
â€¦
group h: [ah - 1, ah - 1 + h, ah - 1 + 2h, â€¦]

Then we decrease the value of h to h1 (h1 < h) and do the same progress again and so on until hn = 1 (when hn = 1, we do insertion sort at last).
But why this works better than insertion sort? Beginning with large values of h, this rearrangement allows elements to move long distances in the original list, reducing large amounts of disorder quickly, and leaving less work for smaller h steps to do. For each group, we do insertion sort which will reduce the average number of shifts from n / 2 to n / 2h. Choosing a good sequence of [h, h1, â€¦, hn] is important for shell sort. Analysis

We start with a large h value and reduce to 1 at last, for each iteration (walk through the list), we divide the list into h groups and do INSERTION SORT for each group.

Example

Example list of numbers:

[5, 19, 15, 41, 7, 1, 11, 25, 8, 3, 32, 2]

Starts with h = 9 (sub number indicates the group number):

[51, 192, 23, 414, 75, 16, 117, 258, 89, 31, 322, 153]

and for each group, we do insertion sort, then after the first iteration:

[31, 192, 23, 414, 75, 16, 117, 258, 89, 51, 322, 153]

Reduce h to 8 (sub number indicates the group number):

[31, 192, 23, 414, 75, 16, 117, 258, 81, 52, 323, 154]

After the second iteration:

[31, 52, 23, 154, 75, 16, 117, 258, 81, 192, 323, 414]

And so on, the list changes as following:
h = 6

[31, 52, 23, 154, 75, 16, 111, 252, 83, 194, 325, 416]

h = 4

[31, 12, 23, 154, 71, 52, 113, 254, 81, 192, 323, 414]

h = 3

[31, 12, 23, 111, 72, 53, 151, 252, 83, 191, 322, 413]

h = 2

[21, 12, 31, 52, 71, 112, 81, 192, 151, 252, 321, 412]

h = 1

[1, 2, 3, 5, 7, 8, 11, 15, 19, 25, 32, 41]

Finished!
